# Architektur

EasyAlias besteht aus zwei Schichten:

- Frontend in `src/main.ts`
- Tauri/Rust Backend in `src-tauri/src/main.rs`

## Grundidee

Die App editiert nicht direkt deine komplette `~/.zshrc`. Stattdessen verwaltet sie eigene Dateien unter:

```text
~/.easyalias/
```

Das reduziert das Risiko, bestehende Shell-Konfigurationen wie `PATH`, Java, Homebrew oder andere Exports kaputt zu machen.

## Datenfluss

```text
UI Formular
  -> AliasEntry
  -> config.json
  -> aliases.zsh
  -> zsh source
```

Im Browser-Dev-Modus ohne Tauri wird der Zustand nur in `localStorage` gespeichert. Das ist absichtlich so, damit die UI auch ohne Rust schnell getestet werden kann.

## Frontend

Das Frontend ist bewusst schlicht gehalten:

- kein Framework
- TypeScript
- Vite als Dev-Server und Build-Tool

Wichtige Aufgaben:

- Alias-Formular verwalten
- Alias-Namen validieren
- Command-Preview erzeugen
- Liste rendern
- Tauri Commands aufrufen, sobald die App in Tauri laeuft

Die wichtigsten Typen sind:

```ts
type AliasAction =
  | "navigate"
  | "open"
  | "execute"
  | "compile_gradle"
  | "compile_maven"
  | "custom";

type AliasEntry = {
  id: string;
  name: string;
  path: string;
  action: AliasAction;
  customCommand?: string;
  commandPreview: string;
  createdAt: string;
  updatedAt: string;
};
```

## Backend

Das Tauri-Backend stellt aktuell zwei Commands bereit:

```rust
load_aliases()
save_aliases(aliases)
```

`load_aliases` liest `~/.easyalias/config.json`, falls die Datei existiert.

`save_aliases` schreibt:

- `config.json` als strukturierte Datenbasis fuer die UI
- `aliases.zsh` als generierte Shell-Datei

## Shell-Generierung

Die generierte Datei sieht z.B. so aus:

```zsh
# Generated by EasyAlias.
# Edit aliases in the app, not by hand.

alias beerv2='cd "$HOME/Desktop/projekte/beerv2_app"'
```

Alias-Namen werden im Frontend und Backend validiert. Das Backend verwirft leere Befehle und ungueltige Namen vor dem Schreiben.

## Sicherheitsnotizen

- Die App sollte langfristig bestehende `~/.zshrc`-Inhalte nur nach Backup oder expliziter Bestaetigung veraendern.
- Custom Commands sind maechtig und sollten in der UI klar als frei ausfuehrbarer Shell-Befehl markiert werden.
- Pfade werden fuer die Standardaktionen in doppelte Anfuehrungszeichen gesetzt.
- Die generierte `aliases.zsh` sollte als Output-Datei gelten und nicht per Hand editiert werden.
